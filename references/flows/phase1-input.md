# 第一阶段：五项核心输入

> **设计理念**：用户只需提供五项核心输入（开篇背景、故事结局、卷纲、能力路径、核心矛盾），Agent根据这五项输入自主黑箱生成全部详细设定。无需用户逐一确认世界观、人物、时间线等细节。

---

## 输入一：开篇背景

使用 `AskUserQuestion` 工具询问：

```
Question: 故事的开篇背景是什么？（世界、时代、初始情境）
Options:
- 悬疑推理（当代都市/封闭空间/连环案件）
- 言情（现代都市/古代宫廷/穿越时空）
- 奇幻玄幻（异世界/修真大陆/魔法文明）
- 科幻未来（星际/末世/赛博朋克）
- 武侠仙侠（江湖/门派/修仙界）
- 历史架空（特定朝代/虚构王朝/权谋朝堂）
- 自由描述（我有完整的世界构想）
```

**记录**：`开篇背景 = [用户选择/描述]`

如用户选择自由描述，完整记录其世界设定、时代背景、初始情境、核心矛盾等。

---

## 输入二：故事结局

使用 `AskUserQuestion` 工具询问：

```
Question: 故事最终走向和结局是怎样的？
Options:
- 大团圆结局（主角成功，众人归位）
- 悲剧结局（主角牺牲/遗憾收场）
- 开放式结局（留有余韵和想象空间）
- 反转结局（出人意料的真相/逆转）
- 苦涩胜利（代价沉重的胜利）
- 自由描述（我已想好结局）
```

**记录**：`故事结局 = [用户选择/描述]`

追问（可选）：
```
Question: 结局时主角的状态是？（可跳过，Agent自行安排）
Options:
- 登顶巅峰/功成名就
- 归隐田园/平淡幸福
- 牺牲自我/成全他人
- 看破红尘/超然物外
- 留下遗憾/未完待续
- 自由描述
```

---

## 输入三：卷纲

使用 `AskUserQuestion` 工具询问：

```
Question: 计划分几卷/几章？各卷的大体内容和结尾效果是？
Options:
- 10章短篇（约3-5万字，1卷）
- 20章中篇（约6-10万字，2-3卷）
- 30章中长篇（约9-15万字，3-4卷）
- 50章长篇（约15-25万字，5-6卷）
- 自定义章节数和卷数
```

**记录**：`总章数 = [用户选择]`

追问卷纲详情：
```
Question: 请描述各卷的大体内容和卷末效果（简要即可，Agent会自动细化）
示例格式：
  第一卷（第1-10章）：主角初入江湖，卷末发现身世秘密
  第二卷（第11-20章）：拜师学艺，卷末与宿敌首次对决
  第三卷（第21-30章）：终极对决，卷末大结局
请自由描述你的卷纲：
```

**记录**：`卷纲 = [用户描述的各卷内容和卷末效果]`

---

## 输入四：主角能力/修为提升路径

使用 `AskUserQuestion` 工具询问：

```
Question: 主角变强的方式和路径是怎样的？（能力/修为提升的核心逻辑）
Options:
- 功法修炼（仙侠/玄幻体系，逐级突破境界）
- 科技研发（科幻体系，解锁技术/装备）
- 异能觉醒（超能体系，逐步开发潜能）
- 资源积累（经营体系，靠财富/势力/人脉成长）
- 血脉觉醒（天赋体系，激发隐藏血脉/体质）
- 混合型（多轨并行，功法+科技/异能+资源等）
- 自由描述（我有完整的能力成长构想）
```

**记录**：`能力路径 = [用户选择/描述]`

追问（可选）：
```
Question: 主角成长过程中的核心代价是什么？（可跳过，Agent自行安排）
Options:
- 生命/寿元消耗
- 亲情/友情/爱情牺牲
- 记忆/情感/人性流失
- 身体/灵魂损伤
- 道德/立场代价
- 自由描述
```

**记录**：`成长代价 = [用户选择/描述]`

---

## 输入五：故事全局核心矛盾

使用 `AskUserQuestion` 工具询问：

```
Question: 贯穿全书的根本矛盾/冲突是什么？（故事的核心驱动力）
Options:
- 正邪对立（光明vs黑暗/正义vs邪恶）
- 生存危机（末日/灾难/种族存亡）
- 权力争夺（王位/资源/势力争霸）
- 身份困境（隐藏身份/身世之谜/阶级跨越）
- 信念冲突（理想vs现实/自由vs秩序）
- 情感纠葛（宿命羁绊/虐恋/背叛与救赎）
- 自由描述（我已想好核心矛盾）
```

**记录**：`核心矛盾 = [用户选择/描述]`

追问（可选）：
```
Question: 核心矛盾在故事中如何逐步升级？（可跳过，Agent自行安排）
Options:
- 个人层面 → 势力层面 → 世界层面（逐步扩大）
- 持续对峙 → 短暂缓和 → 终极爆发（波浪式推进）
- 隐藏真相 → 逐步揭露 → 全盘引爆（悬念驱动）
- 自由描述
```

**记录**：`矛盾升级 = [用户选择/描述]`

---

## 输入完成 → 展示确认

五项输入全部完成后，展示摘要：

```markdown
## 核心输入确认

🌍 **开篇背景**：[用户输入的背景描述]
🎬 **故事结局**：[用户输入的结局描述]
📚 **卷纲规划**：
  [用户输入的各卷内容和卷末效果]
⚔️ **能力路径**：[用户输入的主角变强方式]
🔥 **核心矛盾**：[用户输入的贯穿全书的根本冲突]
```

使用 `AskUserQuestion` 请求确认：

```
Question: 以上是你的五项核心创作方向，确认后我将自动生成全部设定（世界观、人物卡、大纲体系等），无需你逐项确认细节。
Options:
- 确认，开始自动生成！
- 我想修改某些内容
- 我想补充更多细节
```

用户确认后 → 进入第二阶段：黑箱生成全部设定。
