# 第二阶段：黑箱生成全部设定 + 确认

> **设计理念**：用户只需提供开篇背景、故事结局、卷纲三项核心输入。本阶段Agent根据这三项输入自主黑箱生成全部基础设定文件，无需用户逐一确认世界观、人物、时间线等细节。

---

## 执行步骤

### 步骤1：创建项目文件夹

`./chinese-novelist/{YYYYMMDD-HHmmss}-{小说名称}/`

目录结构：
```
{项目文件夹}/
├── 00-世界观.md
├── 01-人物卡/
│   ├── 主角.md
│   ├── 重要角色.md
│   ├── 次要角色.md
│   ├── 重要反派.md
│   └── 次要反派.md
├── 02-大纲/
│   ├── 卷纲.md
│   ├── 时间线.md
│   ├── 伏笔布局.md
│   ├── 进阶条件.md
│   ├── 成长体系.md
│   └── 境界功法.md
└── 03-写作计划.json
```

### 步骤2：生成核心世界观

创建 `00-世界观.md`，根据用户的开篇背景输入，参考 [plot-structures.md](../guides/plot-structures.md) 黑箱生成完整世界观：

- 世界基础规则（物理法则、能量体系、社会结构）
- 势力划分与阵营对立
- 核心冲突根源
- 世界地域架构
- 底层力量/科技体系规则
- 世界观壁垒与禁忌
- 历史背景与重要事件

### 步骤3：生成五类角色档案

创建 `01-人物卡/` 文件夹，根据开篇背景+故事结局，参考 [character-building.md](../guides/character-building.md) 和 [character-template.md](../guides/character-template.md) 黑箱生成五类角色：

**`01-人物卡/主角.md`**
- 完整三维人设：原生性格、成长蜕变线、身世秘密、核心执念、致命软肋
- 专属优缺点、心境变化周期、完整人物弧光
- 成长代价、最终宿命（与故事结局呼应）
- 说话风格/口头禅、行为模式

**`01-人物卡/重要角色.md`**
- 核心配角/羁绊角色：深度立体人设
- 独立支线、个人执念与软肋
- 与主角强羁绊（亲情/友情/爱情/师徒情）
- 专属剧情高光与遗憾、独立结局闭环

**`01-人物卡/次要角色.md`**
- 功能性配角：人设清晰、定位明确
- 服务主线/支线剧情，拥有基础性格、身份、行为逻辑
- 出场合理、退场自然

**`01-人物卡/重要反派.md`**
- 绝对立体反派：独立身世、无奈、执念、信仰
- 与主角形成宿命对立，完整成长线与陨落逻辑
- 反派弧光完整，杜绝无脑坏

**`01-人物卡/次要反派.md`**
- 阶段性敌对势力/小人物反派
- 动机明确，因利益、立场、偏见产生冲突
- 用于铺垫阶段性矛盾、衬托主线冲突

### 步骤4：生成卷纲

创建 `02-大纲/卷纲.md`，将用户的卷纲输入细化完善：

- 保留用户原始的卷章划分和各卷结尾效果
- 细化每卷的核心剧情线、转折点、情绪曲线
- 确定每章的核心事件和悬念钩子
- 整合五类角色的出场安排和人物弧光节点
- 输出完整的章节规划表

### 步骤5：生成时间线管理体系

创建 `02-大纲/时间线.md`，参考 [plot-control-dashboard.md](../guides/plot-control-dashboard.md) 黑箱生成：

- 全书唯一线性时间轴
- 精准锚定所有剧情事件、人物相遇、实力突破、势力更迭的时间节点
- 前后时间完全自洽，无时间悖论
- 所有支线、人物成长严格贴合主线时序

### 步骤6：生成伏笔布局体系

创建 `02-大纲/伏笔布局.md`，参考 [hook-techniques.md](../guides/hook-techniques.md) 黑箱生成：

- 独立虐点/意难平伏笔库
- 预埋误会、遗憾、错过、牺牲、羁绊破裂等隐性刀子伏笔
- 伏笔追踪表：伏笔ID、埋设章节、内容、类型、预计揭晓、状态
- 浅层自然融入日常剧情，支持阶段性铺垫、中段发酵、后段精准回收

### 步骤7：生成进阶条件体系

创建 `02-大纲/进阶条件.md` 黑箱生成：

- 全书所有人物成长、实力突破、心境蜕变、剧情转折、阵营反转的硬性条件+软性契机
- 硬性条件：积累、阅历、资源、劫难等量化基础
- 软性契机：情绪、羁绊、顿悟、绝境触发
- 所有进阶必有铺垫、必有代价、必有成长反馈

### 步骤8：生成成长体系树

创建 `02-大纲/成长体系.md` 黑箱生成：

- 根据题材自动匹配（仙侠玄幻→功法修为树，都市科幻→科技研发树）
- 体系树层级清晰、分支明确、进阶路径唯一且闭环
- 包含基础层级、进阶层级、巅峰层级、隐藏特殊层级
- 每一级进阶对应专属能力、短板、副作用、战力变化

### 步骤9：生成境界功法体系

创建 `02-大纲/境界功法.md` 黑箱生成：

- 完整的境界划分（如炼气→筑基→金丹→元婴→化神...）
- 每个境界的能力特征、突破条件、战力标准
- 功法分类与品阶体系
- 特殊体质/血脉/天赋系统
- 丹药/法宝/阵法等辅助体系（如适用）

### 步骤10：生成写作计划

创建 `03-写作计划.json`，基于卷纲自动生成：

```json
{
  "version": 1,
  "novelName": "[小说名称]",
  "projectPath": "./chinese-novelist/{timestamp}-[小说名称]",
  "totalChapters": [章节数],
  "minWordsPerChapter": 3000,
  "createdAt": "[ISO时间]",
  "updatedAt": "[ISO时间]",
  "status": "planning",
  "writingMode": null,
  "chapters": [
    {
      "chapterNumber": 1,
      "title": "[章节标题]",
      "filePath": "第01章-[章节标题].md",
      "status": "pending",
      "wordCount": null,
      "wordCountPass": null,
      "retryCount": 0
    }
  ]
}
```

## 完成后：展示规划摘要并请求确认

向用户展示自动生成的规划摘要：

```markdown
## 自动生成的创作规划

🌍 **世界观**：[一句话概述世界设定]
👥 **五类角色**：
  主角：[名字/一句话人设]
  重要角色：[名字列表]
  重要反派：[名字/一句话人设]
📚 **卷纲结构**：[X]卷 [Y]章
⏳ **时间线**：[时间跨度概述]
🗡️ **伏笔数量**：[X]条预埋伏笔
📈 **成长体系**：[体系类型] [X]个大境界
```

使用 `AskUserQuestion` 请求确认：

```
Question: 以上是根据你的核心输入自动生成的全部设定，确认后开始创作。
Options:
- 确认，开始创作！
- 我想查看/修改某个具体设定
- 重新生成全部设定
```

用户确认后 → 选择写作模式 → 进入第三阶段。

---

## 写作模式选择

使用 `AskUserQuestion` 询问：

```
Question: 选择写作模式
Options:
- 逐章串行（主 Agent 自己逐章写，全程无中断，适合短中篇）
- 子Agent并行（分批派生子 Agent 并行写作，大纲驱动连贯性，适合中长篇）
- Agent Teams（Claude Code 多 Agent 协作模式，Agent 间可通讯，需手动开启）
```

用户选择后：
- 更新 `03-写作计划.json` 的 `writingMode` 字段
- 更新 `status` 为 `"in_progress"`
- 进入第三阶段：疯狂创作 -> 详见 [phase3-writing.md](phase3-writing.md)
