# 第四阶段：自动校验与修复

**目标**：确保所有章节完成且字数达标，同时检查伏笔回收完整性、时间线自洽、人设一致性，全程无需用户介入。

---

## 校验流程

### 1. 初始化校验

读取 `03-写作计划.json`，将项目 `status` 更新为 `"validating"`。

### 2. 逐章检查

对每一章执行以下检查：

1. **文件存在性**：检查 `filePath` 指定的文件是否存在
2. **字数检查**：使用脚本检查字数
   ```bash
   python scripts/check_chapter_wordcount.py <章节文件路径>
   ```
3. **更新 JSON**：将 `wordCount` 和 `wordCountPass` 写入对应章节记录
4. **章节衔接检查**：
   - [ ] 本章开篇是否承接了上章结尾的悬念？
   - [ ] 时间点与上章结尾是否连续？
   - [ ] 人物的位置和情绪状态与上章一致？

### 3. 完整性检查

#### 3.1 伏笔完整性
- [ ] 伏笔追踪表中所有伏笔均已回收或标记为"有意留白"
- [ ] 无遗漏的"孤儿伏笔"（埋设后永远不回收）
- [ ] 刀子伏笔：前期的铺垫在对应章节有呼应/回收
- [ ] 没有"天降解答"（回收时出现未埋设的新信息）

#### 3.2 时间线自洽性
- [ ] 日期连续，无跳跃错误
- [ ] 时间段合理，角色无分身（同一时间在两个地方）
- [ ] 季节/天气与日期匹配
- [ ] 角色年龄与时间流逝一致
- [ ] 人物进阶时间点合理（不会凭空突破）

#### 3.3 人设一致性
- [ ] 人物行为符合其性格设定，不突兀
- [ ] 台词风格辨识度一致
- [ ] 人物关系变化有迹可循
- [ ] 角色成长有前置条件铺垫

#### 3.4 成长体系验证
- [ ] 主角进阶条件是否按大纲要求触发？
- [ ] 硬性条件已满足，软性契机已出现？
- [ ] 每次突破有相应的成长代价描写？
- [ ] 成长体系树的每一级进阶都已使用？

### 4. 文风质量检查

#### 4.1 AI高频废词检查
搜索并标注以下词汇的出现次数，超过5次的章节标记为不合格：
- "此刻、见状、随即、不由得、不禁"
- "仿佛、宛如、犹如、恰似"
- "油然而生、心潮澎湃、热血沸腾"
- "彰显、诠释、赋能、映射"

#### 4.2 句式多样性检查
- [ ] 无连续3句主语相同的短句
- [ ] 长短句交替，无通篇长句
- [ ] 段落长度变化，无大段密集文字堆砌
- [ ] "的"字密度控制（一句话不超过2个"的"）

#### 4.3 展示而非讲述检查
- [ ] 情绪通过动作表现，而非直接陈述
- [ ] 环境通过感官描写呈现，而非概括性描述
- [ ] 人物性格通过行为展示，而非标签式说明

### 5. 汇总结果

- **全部通过**（所有章节文件存在且 `wordCountPass == true`，且完整性和文风检查均通过）-> 更新项目 `status` 为 `"completed"`，进入步骤7
- **有不通过** -> 进入步骤6

### 6. 自动修复（最多3轮）

对每个不通过的章节：

1. 将 `status` 设为 `"failed"`，`retryCount` 加 1
2. 回到第三阶段对该章节执行逐章创作流程（写前分析->撰写->润色->收尾），**重点关注未通过项**
3. 重写完成后重新检查字数、文风和完整性
4. 更新 `03-写作计划.json`

循环规则：
- 最多执行 **3轮** 校验-修复循环
- 超过3次仍不合格的章节，保留记录但不阻塞流程
- 所有修复完成后更新项目 `status` 为 `"completed"`

### 7. 完成报告

向用户展示创作完成总结：

```

总章数：[X] 章
总字数：[X] 字
完成率：[X]%

各章节状态：
✅ 第1章：[标题]（[字数]字）
✅ 第2章：[标题]（[字数]字）
...

伏笔回收：[X]/[X]（[X]%）
时间线连续性：✅/❌
人设一致性：✅/❌
文风质量：✅/❌

```

如有不合格章节（超过3次重写仍未通过），在报告中标注：
```

⚠️ 第X章：文风中检测到AI高频词 [次数] 次
```
