### 一、人物角色指引体系增强

#### 1. 更新 `references/guides/character-building.md`

**新增内容：**
- **四类角色创作规范**章节，详细定义：
  
  **① 主角 (Protagonist)**
  - 核心要素：明确目标、强大动机、可共情性、成长空间
  - 性格弧光设计：起始状态 → 触发事件 → 抗拒改变 → 被迫面对 → 突破临界点 → 新状态
  - 成长路径规划：分阶段写作要点表格
  - 主角原型与适配：英雄型/成长型/反英雄型/平凡型
  - 缺陷致命化：5种常见缺陷及其导致的失败和成长方向
  
  **② 重要配角 (Key Supporting Characters)**
  - 功能性关系定义表：导师型/盟友型/镜像型/催化剂型/阻碍型
  - 每种类型的核心特征、写作要点、示例
  - 配角独立性检查清单（4项测试）
  
  **③ 反派 (Antagonist)**
  - 三层动机模型：表层动机 → 中层动机 → 深层动机
  - 反派与主角的镜像关系对比结构
  - 反派塑造标准（5项）
  - 反派类型分类：理念型/个人型/系统型/内在型/自然型
  - 反派出场与退场原则
  
  **④ 其他角色 (Minor Characters)**
  - 快速构建三要素法则：鲜明特征 + 明确功能 + 微小动机
  - 避免工具人化的技巧对比表
  - 功能性配角分类：信息提供者/场景营造者/冲突制造者/情感触发者
  - 群像戏处理策略

**改进效果：**
- 从通用的"主角/反派/配角"三分法升级为更精细的四类角色体系
- 每类角色都有明确的构建方法和检查标准
- 提供了大量实例和对比表格，便于实际操作

  - 基础信息：年龄/职业、外貌特征、说话风格/口头禅
  - 核心驱动：内心渴望、外在目标、强大动机、核心价值观
  - 性格弧光：起始状态、致命缺陷、最大恐惧、成长方向、突破临界点
  - 背景故事：关键经历、创伤/遗憾、隐藏秘密
  - 关系网络：与反派/盟友/导师的关系
  - 行为模式：压力下的反应、习惯性动作、决策风格
  - MBTI/九型人格（可选）
  
  **反派模板**（完整）：
  - 基础信息：年龄/职业、外貌特征、说话风格
  - 动机层次：表层/中层/深层动机、自认为的正义
  - 与主角的镜像关系：共同起点、分岔路口、黑暗面展示、理念冲突点
  - 能力与资源：核心能力、资源网络、致命弱点、盲点
  - 背景故事：堕落轨迹、创伤源头、同情点
  - 退场设计：失败原因、最后时刻、遗留影响
  
  **重要配角模板**（简化）：
  - 基础信息：年龄/职业、外貌特征、说话风格
  - 功能性定位：与主角的关系、对主角的作用、独立动机、与主线的关联
  - 个人弧光（如有）：起始/成长/结局状态
  - 关键场景：首次出场、高光时刻、退场方式
  - 背景简述：核心经历、秘密/隐情
  
  **其他角色模板**（最简）：
  - 快速构建三要素：鲜明特征、明确功能、微小动机
  - 出场信息：出现章节、出场场景、关键台词/动作

- **角色关系图谱**（可选）：可视化主要角色之间的关系线
- **角色清单检查**：7项检查清单

**改进效果：**
- 从单一的通用模板升级为四类差异化模板
- 每个模板的字段都针对该类型角色的特点设计
- 提供了从完整到简明的梯度，适应不同重要程度的角色

### 二、剧情控制台模块新增

#### 1. 新建 `references/guides/plot-control-dashboard.md`（520行）

**核心内容：五大维度管理方法**

**① 卷纲 (Volume Outline)**
- 卷的结构设计：三幕结构模板
- 卷间衔接技巧：悬念延续/时间跳跃/视角转换/地点转移/主题升级
- 卷纲模板：包含基本信息、三幕结构、关键角色、伏笔清单
- 多卷主题递进示例

**② 章纲 (Chapter Outline)**
- 七列详解表格：填写要点和检查标准
- 章节节奏控制表：🔴快/🟡中/🟢慢标记系统
- 章节类型分布：情节推进章50-60%、人物深化章20-25%等
- 节奏原则：避免连续3章相同节奏、高潮前后应有缓冲

**③ 时间线 (Timeline)**
- 单一时间线管理：时间线表格模板、检查要点
- 多线叙事时间线：并行时间线对照表、切换规则
- 倒叙/插叙管理：触发点合理性、标记方法
- 时间悖论检查（科幻/奇幻类）

**④ 伏笔管理 (Foreshadowing Management)**
- 伏笔追踪表模板：伏笔ID、埋设章节、内容、类型、预计/实际揭晓、状态
- 伏笔类型分类：物品/对话/行为/环境/人物伏笔
- 草蛇灰线技法：埋设原则（伪装成闲笔、多次重复、渐进揭示、回收时震撼）
- 伏笔密度控制：前期密集埋设、中期开始回收、后期大量回收
- 伏笔与悬念的区别对比表

**⑤ 视角表 (POV Chart)**
- 视角类型定义：第一人称/第三人称限制/全知/第二人称
- 多视角切换规则：黄金法则（一章一视角）、明确标记、避免中段切换
- 视角追踪表模板
- 视角跳跃错误示例对比（错误vs正确）
- 视角选择策略：何时切换/不切换
- POV一致性检查清单
- 特殊视角技巧：不可靠叙述者、多重视角重复事件、旁观者视角

**使用流程：**
- Phase 2 规划阶段：建立卷纲、完善章纲、创建时间线、初始化伏笔表、确定视角策略
- Phase 3 写作阶段：每章写作前查阅、写作后更新
- Phase 4 校验阶段：全局检查、修复问题

**工具推荐：**
- 电子表格管理（Excel/Notion/Airtable）
- 可视化时间线工具（Timeline JS/Aeon Timeline/Miro）
- 版本控制（Git跟踪变更）

**常见问题解答：**
- Q1: 伏笔太多记不住怎么办？
- Q2: 多视角会不会让读者混乱？
- Q3: 时间线出错怎么修复？
- Q4: 卷与卷之间如何保持连贯？
- Q5: 章纲太详细会不会限制创作自由？

## 🔧 使用建议

### 对于新用户

1. **先阅读 `plot-control-dashboard.md`**：了解五大维度的基本概念和使用方法
2. **Phase 2 规划时认真填写**：卷纲、章纲、时间线、伏笔表、视角表都要仔细规划
3. **Phase 3 写作时严格遵循**：每章写前查阅控制台，写后及时更新
4. **善用检查清单**：每个维度都有检查清单，定期自查

### 对于经验丰富的用户

1. **灵活调整**：剧情控制台是指南而非枷锁，根据实际情况调整
2. **电子表格辅助**：复杂项目建议使用 Excel/Notion 管理伏笔追踪表等
3. **版本控制**：使用 Git 跟踪大纲和控制台的变更，便于回溯
4. **定期审查**：每5-10章审查一次控制台，确保没有遗漏
